# Design Tokens

Motive's design system is a **three-layer token model**:

1. **Primitive palette** (`mtvColors`) — the raw hex source of truth, exported from
   `@motive/tailwind-config`. The runtime CSS layer uses the oklch equivalents.
2. **Semantic tokens** (`--mtv-color-*` CSS custom properties) — purpose-named roles
   (foreground, surface, border, brand, status, chart, link) that map onto primitives and
   shift per `data-theme` (Arc / Console / Legacy) and the Arc hue/tint controls.
3. **Component usage** — components reference the semantic tokens, never primitives directly,
   so a theme switch re-skins everything consistently.

Everything below is generated programmatically from the exported `mtvColors` and `tokens`
objects (and the semantic CSS variables), so it stays in sync with the source. Use the theme
and Arc tint toolbar controls to watch the **semantic** swatches react.
